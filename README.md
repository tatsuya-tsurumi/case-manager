# CaseManager

Spring Boot + MyBatis を用いた **ケース管理／タスク管理アプリケーション**です。  
実務を想定したレイヤードアーキテクチャと、拡張しやすいデータベース設計を意識して開発しています。

---

## 概要

CaseManager は、以下のような業務を想定した管理アプリです。

- ケース（案件）の登録・一覧表示・詳細確認
- ケースに紐づくタスクの管理
- ステータス・優先度をコード管理することで実務向けの設計を実現

基本機能は動作確認済みです。

## データベース設計

### User テーブル
| カラム名 | 内容 |
|--------|------|
| user_id | ユーザID（PK, auto increment） |
| email | メールアドレス |
| user_name | ユーザ名 |
| password | パスワード（ハッシュ） |

### Case テーブル
| カラム名 | 内容 |
|---------|------|
| case_id | ケースID |
| user_id | 担当ユーザID |
| case_name | ケース名 |
| client_name | クライアント名 |
| status_code | ステータスコード |
| detail | 内容 |

### Task テーブル
| カラム名 | 内容 |
|---------|------|
| task_id | タスクID |
| case_id | ケースID |
| title | タイトル |
| deadline | 期限 |
| priority_code | 優先度コード |
| status_code | ステータスコード |

### Status テーブル
| カラム名 | 内容 |
|------------|------|
| status_code | ステータスコード（00,05,10…） |
| status_name | ステータス名 |

### Priority テーブル
| カラム名 | 内容 |
|---------------|------|
| priority_code | 優先度コード（00,05…） |
| priority_name | 優先度名（低・中・高） |

## 設計上のポイント

- **Controller / Service / Repository を明確に分離**
- **ビジネスロジックは Service 層に集約**
- **MyBatis を使用し SQL を明示的に管理**
- **1対多の関係は resultMap + collection で表現**
- **ステータス・優先度はコード管理で拡張性を確保**
